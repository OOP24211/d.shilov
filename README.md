# Echo Messenger

Мессенджер на JavaFX + WebSocket + PostgreSQL. Полностью переписан с нуля,
новый интерфейс (тёмная тема с фиолетовым акцентом) и новая функциональность.

## Возможности

- Регистрация и вход (пароли хранятся в виде SHA-256 хеша, не в открытом виде)
- Личные чаты «один на один» и групповые чаты
- Поиск пользователей и групп по нику (двойной клик → открыть/создать)
- В группы могут писать все участники; добавление участников, выход из группы, список участников
- Статусы онлайн/оффлайн — зелёные точки в списке и в шапке чата
- Аватарки с инициалом и цветом, выведенным из ника
- Счётчики непрочитанных сообщений (бейджи в списке чатов)
- Превью последнего сообщения в списке чатов
- Индикатор «печатает…»
- Удаление своих сообщений (правый клик по сообщению → Удалить)
- Разделители по датам в переписке

## Настройка PostgreSQL

```sql
CREATE USER echo_user WITH PASSWORD 'echo_pass';
CREATE DATABASE echo_db OWNER echo_user;
GRANT ALL PRIVILEGES ON DATABASE echo_db TO echo_user;
```

Таблицы (`users`, `chats`, `members`, `messages`, `reads`) создаются автоматически
при первом запуске сервера. Если хочешь другие имя БД / логин / пароль —
поправь константы вверху `Db.java`.

## Запуск

### Вариант A — через IntelliJ IDEA
1. Открыть папку `echo-messenger` как Maven-проект (IDEA сама подтянет зависимости).
2. Запустить PostgreSQL.
3. Запустить `app.echo.server.ServerMain` (метод main) — сервер на порту 9696.
4. Запустить `app.echo.ui.App` (метод main) — окно входа.

Можно запустить несколько копий `App` одновременно — проверить чат вживую
(онлайн-статусы, «печатает…», доставку сообщений).

### Вариант B — через Maven из терминала
```bash
# 1) сервер
mvn -q compile exec:java -Dexec.mainClass="app.echo.server.ServerMain"
# (или просто запусти ServerMain в IDE)

# 2) клиент (в отдельном окне терминала)
mvn -q javafx:run
```

> Если при запуске клиента ругается на модули JavaFX — проще запускать из IntelliJ,
> там JavaFX подключается автоматически из Maven-зависимостей.

## Как пользоваться

| Действие | Как |
|----------|-----|
| Регистрация | Ввести логин (от 3 символов) и пароль → «Создать аккаунт» |
| Вход | Логин/пароль → «Войти» |
| Личный чат | Поиск по нику → двойной клик по пользователю |
| Создать группу | Кнопка «＋» рядом с поиском → ввести название |
| Войти в существующую группу | Поиск по названию → двойной клик |
| Добавить в группу | Открыть группу → «Добавить» в шапке → ввести логин |
| Список участников | Кнопка «Участники» в шапке (онлайн помечены зелёным) |
| Покинуть группу | Кнопка «Покинуть» в шапке |
| Удалить своё сообщение | Правый клик по сообщению → «Удалить» |
| Отправить | Enter или кнопка ➤ |
| Выход из аккаунта | Кнопка «Выйти» вверху слева |

## Заметки по архитектуре

- Обмен между клиентом и сервером — JSON-объекты `Packet` (Jackson) поверх WebSocket.
- Сервер хранит онлайн-сессии в памяти (`Map<WebSocket, login>`) и рассылает
  `PRESENCE` всем при входе/выходе любого пользователя.
- Непрочитанные считаются через таблицу `reads` (last_read_id на пару чат+юзер);
  при открытии чата отметка читается автоматически.
- Удаление сообщения проверяется на сервере: удалить можно только своё.
- Порт сервера — 9696 (меняется в `ServerMain` и в `AuthController` адресе `ws://localhost:9696`).
